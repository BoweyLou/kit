# Agentic Workflow Spikes

Spike notes in this directory capture bounded research for backlog items that
should not ship runtime behavior in the same change.

Each note should name the question, local evidence inspected, external sources
used if any, recommendation, stop conditions, architecture sketch, validation
plan, and the explicit follow-on backlog shape. If a spike recommends later
implementation, keep shipped code, commands, schemas, prompts, and adapters out
of the spike change unless the task packet explicitly allows them.
