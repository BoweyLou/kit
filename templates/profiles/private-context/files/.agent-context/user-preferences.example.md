# User Preference Context Example

Use this example for non-sensitive local workflow preferences that help an agent
collaborate with you. Keep it short and avoid biographical detail.

Do not include secrets, tokens, cookies, passwords, private URLs, account
identifiers, customer data, personal messages, medical or financial data, or
proprietary snippets that should not leave the machine.

Before sharing any copied file with hosted models, browser tools, GitHub
comments, pull requests, issues, external tickets, or chat tools, review and
redact it.

## Safe To Record

- Preferred response length for routine status updates.
- Preferred test command order when the repo offers several equivalent checks.
- Formatting or commit-message preferences that are not sensitive.
- Collaboration preferences that are safe for repo collaborators to know.

## Example Notes

- Keep routine progress updates brief and concrete.
- Prefer local verification commands before remote assumptions.
- Mention skipped checks and the reason they were skipped.
